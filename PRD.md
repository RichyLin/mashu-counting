# 麻薯计分 — 产品需求文档

**版本：** 1.0  
**日期：** 2026-04-25  
**状态：** 交互原型已验证，待开发

---

## 一、产品概述

### 1.1 产品定位

麻薯计分是一款专为**贵州捉鸡麻将**设计的 H5 网页计分工具。玩家在同一局麻将游戏中，通过手机浏览器实时记录分数，避免口头记分混乱。

### 1.2 目标用户

熟悉贵州捉鸡麻将玩法的玩家，以朋友聚会场景为主。

### 1.3 核心价值

- 实时同步：所有玩家看到的分数状态保持一致
- 操作简单：针对手机端设计，单手操作为主
- 有据可查：完整的交易记录，支持撤回

### 1.4 平台

- H5 网页，手机浏览器访问
- 设计基准宽度：375px（iPhone SE）
- 不开发原生 App

---

## 二、名词定义

| 名词 | 说明 |
|------|------|
| 房间 | 一局游戏的虚拟空间，由 4 位数字编号标识 |
| 房主 | 第一个进入房间的玩家，拥有踢人、重置、解散权限 |
| 公池 | 所有玩家共享的分数池，交公池/收公池操作均作用于此 |
| 座位 | 麻将桌上的固定位置：上/左/右/下（下方固定为自己） |
| 转账 | 一名玩家将分数付给另一名玩家的操作 |

## 三、用户流程

```
打开页面
    │
    ▼
① 入口页：输入4位房间号
    │
    ▼
② 身份设置页：选emoji头像 + 输入姓名
    │（同一设备再次进入同一房间，跳过此步直接恢复）
    ▼
③ 主界面：麻将桌
    ├── 点击对方头像 → 转账弹窗
    ├── 点击公池区域 → 公池操作弹窗
    ├── 长按对方头像拖动 → 换座确认
    ├── 底部「记录」→ 历史记录弹窗
    └── 底部「设置」（房主专属）→ 房间设置弹窗
```

## 四、页面详细需求

### 4.1 入口页

**功能：**
- 居中展示产品 Logo（麻薯贱宝正视图 + 产品名「麻薯计分」）
- 副标题「贵州捉鸡麻将」
- 4位数字输入框（仅允许输入数字，满4位自动锁定）
- 「进入房间」按钮
- 提示文案：「房间不存在将自动创建」

**规则：**
- 输入框限制：只允许数字，限4位，不足4位不可提交
- 创建与加入不区分：输入房间号后点击进入，若该房间无人则自动创建，有人则加入
- 若本设备已有该房间的身份记录，跳过身份设置直接进入主界面

---

### 4.2 身份设置页

**功能：**
- 返回按钮（回到入口页）
- 头像选择区：展示 emoji 网格（5列×3行，共15个），点击选中，高亮已选项
- 姓名输入框
- 「进入房间」按钮（姓名为空时禁用）

**规则：**
- 头像默认选中第一个 emoji（🍡）
- 不支持上传照片或拍照，仅支持从预设 emoji 中选择
- 姓名限制：
  - 仅允许汉字、字母、数字，不允许特殊字符（空格、标点等）
  - 最多6个字符
  - 实时校验，不合规时显示错误提示并禁用提交按钮
- 进入后，第一个进入该房间的玩家自动成为房主

**身份持久化：**
- 用 localStorage 绑定设备 ID + 房间号，记录玩家身份
- 同一设备再次访问同一房间，无需重新设置身份，直接恢复上次状态

---

### 4.3 主界面（麻将桌）

**整体布局（从上至下）：**

```
┌─────────────────────┐
│  房间号      在线人数 │  ← 顶部导航栏（56px）
├─────────────────────┤
│         [上]         │
│  [左]  公池  [右]   │  ← 麻将桌区域（flex:1）
│         [下=我]      │
├─────────────────────┤
│    记录  |  设置     │  ← 底部导航栏（68px，设置仅房主可见）
└─────────────────────┘
```

**麻将桌区域：**
- 背景：淡绿色光晕
- 中央：抹茶绿椭圆形桌面（深棕描边）
- 公池区域位于桌面中央（见 4.3.1）
- 4个固定座位：上/左/右/下（见 4.3.2）

#### 4.3.1 公池区域

- 显示：麻薯贱宝侧躺图 + 金色分数框（「公池 X 分」）
- 初始显示「公池 0 分」
- 点击任意区域 → 打开公池操作弹窗
- 公池分数实时同步至所有玩家

#### 4.3.2 玩家座位

每个座位显示：
- 圆形 emoji 头像（其他玩家：深棕描边；自己：金色描边+皇冠图标）
- 姓名标签（白底深棕描边胶囊形）
- 分数（**仅自己的分数对自己可见**，其他玩家分数不显示）

空位规则：
- 不满4人时，空位不显示占位符，直接显示麻将桌背景
- 4个座位位置固定（上/左/右/下），下方始终为本机玩家

自己的座位布局（从上至下）：
1. 圆形头像（金色描边，顶部显示皇冠👑）
2. 姓名标签（「姓名（我）」格式）
3. 分数（Fredoka字体，大号）

#### 4.3.3 换座交互

- 操作方式：**长按（300ms）**其他玩家头像后拖动，拖拽至目标座位或自己头像上松手
- 长按时：源头像变透明，出现跟随手指的 ghost 头像
- 移动超过8px但未满300ms：取消长按，不触发换座
- 松手后：弹出换座确认弹窗（显示双方头像和姓名）
- 确认后：仅交换座位的显示身份（姓名/头像），各自分数不变
- **自己的头像不可拖动**
- 换座操作为**本地操作，不同步给其他玩家**（各自看到的座位布局可不同）

---

### 4.4 转账弹窗（底部 Sheet）

**触发：** 点击（短按，< 300ms）其他玩家头像

**内容：**
- 收款方信息行：emoji头像 + 姓名标签（「付给对方」）
- 提示文案：「你将付出多少分给 XX？」
- 数字输入框（大号，居中）
- 「确认转账」按钮（樱花粉色）

**规则：**
- 方向固定：**我付给对方**（不支持反向，即不支持对方付给我）
- 金额：正整数，无下限（可导致余额为负）
- 确认后：自己分数 -N，对方分数 +N（对方分数仅对方可见）
- 生成一条历史记录：「我 → 对方 N分」

---

### 4.5 公池操作弹窗（底部 Sheet）

**触发：** 点击主界面公池区域

**内容：**
- 当前公池分数展示（金色大号）
- 「交公池（-1分）」按钮（嫩芽绿）
- 「收公池（+X分）」按钮（黄金色）

**规则：**
- 任何玩家均可操作公池（不限房主）
- 交公池：固定扣除操作者 1 分，公池 +1
- 收公池：操作者获得公池全部分数，公池归零
- 公池为 0 时，「收公池」按钮置灰禁用
- 两个操作均可导致余额为负（无限制）
- 操作后生成历史记录

---

### 4.6 历史记录弹窗（底部 Sheet）

**触发：** 点击底部导航「记录」

**内容：**
- 标题「交易记录」
- 按时间倒序排列的记录列表

**每条记录格式：**
```
阿福 → 老虎  -5    [撤回]
```
- 正数（收入）：绿色
- 负数（支出）：红色
- 公池相关：金黄色
- 已撤回记录：灰色，无撤回按钮，显示「已撤回」

**撤回规则：**
- 点击「撤回」→ 弹出二次确认弹窗
- 确认后：
  1. 原记录标记为「已撤回」（不删除，保留在列表中）
  2. 新增一条【撤回】记录在列表顶部（分数变动反向）
- 【撤回】记录本身不可再次撤回

---

### 4.7 房间设置弹窗（底部 Sheet，仅房主）

**触发：** 点击底部导航「设置」

**内容与规则：**

| 功能 | 说明 |
|------|------|
| 初始分数 | 可编辑，影响重置后各玩家的起始分数 |
| 踢出玩家 | 列出所有其他玩家，每人有「踢出」按钮，需二次确认 |
| 重置房间 | 所有玩家分数恢复初始值，清空历史记录，需二次确认 |
| 解散房间 | 清除所有数据，所有玩家退出房间，需二次确认，返回入口页 |

## 五、房间生命周期

### 5.1 创建

- 任意玩家输入4位房间号进入，若房间不存在则自动创建
- 第一个进入的玩家自动成为房主

### 5.2 加入

- 其他玩家输入同一房间号进入，最多4人

### 5.3 自动释放

满足以下任一条件时，房间自动释放（数据清除）：
- 所有玩家离线超过 **5分钟**
- 最后一次有效操作距今超过 **2小时**（无人操作超时）

### 5.4 手动解散

房主在设置中点击「解散房间」，二次确认后立即解散。

---

## 六、分数规则

- 初始分数：默认200分（房主可在设置中修改）
- 分数无下限：可为负数
- 各玩家分数对自己可见，对其他玩家不可见
- 公池分数对所有人可见

---

## 七、实时同步规则

以下操作需实时同步至所有在线玩家：

| 操作 | 同步内容 |
|------|---------|
| 转账 | 双方分数变化、历史记录 |
| 交/收公池 | 操作者分数、公池分数、历史记录 |
| 撤回 | 相关分数恢复、历史记录更新 |
| 踢出玩家 | 被踢玩家退出，其他人座位更新 |
| 重置房间 | 所有人分数重置、历史记录清空 |
| 解散房间 | 所有人退出 |

**不同步的操作：**
- 换座（本地视图操作，每个玩家可有不同的座位布局）

- 中文：ZCOOL KuaiLe（Q版可爱风格）

| 使用场景 | 形态 | 尺寸 |
|----------|------|------|

- 所有卡片/输入框：深棕色描边（`1.5px solid #4A3728`）

- 二次确认弹窗：居中模态，毛玻璃遮罩
- Toast 提示：底部弹出，1.8秒后自动消失

| 层级 | 技术选型 |

### 数据库表结构（草案）

```
rooms          — id, code, host_device_id, init_score, created_at
players        — id, room_id, device_id, name, emoji, seat, score, online
transactions   — id, room_id, from_player_id, to_player_id, amount, type, revoked, created_at
room_pool      — room_id, score（单行，随房间存在）
```

## 十、非功能需求

- **响应速度：** 实时操作（转账、公池）同步延迟 < 500ms
- **并发：** 单房间最多4人同时操作，无需高并发设计
- **离线降级：** 网络断开时提示，操作禁用，重连后自动恢复
- **兼容性：** iOS Safari 15+、Android Chrome 90+
- **安全：** 无敏感数据，房间号作为简单访问控制（非加密）

---

## 十一、不在范围内（Out of Scope）

- 用户账号/登录体系
- 多局历史回顾
- 游戏规则自动计分（如自动识别胡牌得分）
- 桌面端适配
- 换座同步给其他玩家
- 头像照片上传（仅 emoji 选择）
- 好友系统、社交功能
