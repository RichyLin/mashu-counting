## 1. 入口页

### TC-01 初始状态
- **前置：** 直接打开产品首页
- **预期：** 显示麻薯贱宝 Logo、「麻薯计分」标题、「创建房间」和「加入房间」两个按钮，不显示输入框

### TC-02 创建房间 — 正常流程
- **操作：** 点击「创建房间」→ 输入一个数据库中不存在的4位号码（如 `2345`）→ 点击「创建房间」
- **预期：** 跳转至身份设置页，房间被创建，本设备成为房主

### TC-03 加入房间 — 正常流程
- **前置：** 已有另一设备创建了房间 `1234`
- **操作：** 点击「加入房间」→ 输入 `1234` → 点击「加入房间」
- **预期：** 跳转至身份设置页（若本设备未有该房间身份）

### TC-04 创建房间 — 号码已存在（active）
- **前置：** 房间 `5678` 已存在且 active
- **操作：** 点击「创建房间」→ 输入 `5678` → 确认
- **预期：** 输入框下方内联报错「该房间号已存在，请加入或换一个号码」，不跳转

### TC-05 加入房间 — 号码不存在
- **操作：** 点击「加入房间」→ 输入 `0000`（不存在的号码）→ 确认
- **预期：** 内联报错「该房间号不存在，请检查号码或创建新房间」，不跳转

### TC-06 创建房间 — 号码曾存在但已 dissolved
- **前置：** 房间 `8888` 曾存在，状态为 `dissolved`
- **操作：** 点击「创建房间」→ 输入 `8888` → 确认
- **预期：** 成功创建新房间并跳转身份页（dissolved 旧房间不应阻止同号码复用）

### TC-07 输入框限制 — 非数字字符
- **操作：** 点击「创建房间」→ 在输入框中输入字母或特殊字符
- **预期：** 字符被过滤，输入框只接受数字

### TC-08 输入框限制 — 不足4位提交
- **操作：** 点击「创建房间」→ 输入 `12`（2位）→ 点击确认
- **预期：** 内联报错「请输入4位数字房间号」，不跳转

### TC-09 返回按钮
- **操作：** 点击「创建房间」→ 进入输入状态后点击「‹ 返回」
- **预期：** 回到初始两按钮状态，输入框和错误提示消失

### TC-10 回车键触发提交
- **操作：** 点击「加入房间」→ 输入4位号码 → 按回车键
- **预期：** 等同于点击确认按钮，触发校验和跳转

## 2. 身份设置页

### TC-11 默认状态
- **预期：** 默认选中第一个 emoji（🍡），姓名输入框为空，「进入房间」按钮禁用

### TC-12 Emoji 选择
- **操作：** 点击任意一个 emoji
- **预期：** 顶部头像预览更新为选中 emoji，被选中 emoji 有高亮边框，其余无边框

### TC-13 正常进入房间
- **操作：** 选择 emoji → 输入合法姓名（如 `阿强`）→ 点击「进入房间」
- **预期：** 跳转至主界面，玩家出现在对应座位，分数为初始值（默认200）

### TC-14 姓名为空时禁用
- **操作：** 不输入姓名直接点击「进入房间」
- **预期：** 按钮不可点击（disabled 状态）

### TC-15 姓名含特殊字符
- **操作：** 输入 `阿@强` 或 `a b`（含空格/标点）
- **预期：** 实时报错「仅支持汉字、字母、数字」，按钮禁用

### TC-16 姓名超出6字符
- **操作：** 尝试输入超过6个字符
- **预期：** 输入框最多接受6个字符，第7个字符被截断

### TC-17 第5位玩家加入（备战席提示）
- **前置：** 房间已有4名场上玩家
- **操作：** 第5设备进入身份页
- **预期：** 页面显示「当前桌位已满，你将以备战席身份加入」提示，「进入房间」按钮可用

### TC-18 第7位玩家加入（满员拒绝）
- **前置：** 房间已有6名玩家（4场上 + 2备战）
- **操作：** 第7设备进入身份页
- **预期：** 页面显示「房间已满（最多6人），无法加入」，「进入房间」按钮禁用

### TC-19 回车键触发提交
- **操作：** 输入合法姓名后按回车键
- **预期：** 等同于点击「进入房间」按钮

- **预期：** 所有玩家视角中，房主头像上方显示👑图标

### TC-22 分数仅自己可见
- **前置：** 2名玩家在同一房间
- **预期：** 每人只能看到自己的分数，看不到其他玩家的分数数字

### TC-23 自己的姓名标签格式
- **预期：** 自己的姓名标签显示为「姓名（我）」格式

### TC-24 空位占位符
- **前置：** 房间只有2名场上玩家
- **预期：** 另外2个空座位显示虚线圆圈占位符，无姓名和分数

### TC-25 公池区域显示

### TC-26 备战席显示位置

- **预期：** bench-left 玩家显示在画布左上角，bench-right 显示在右上角，头像有虚线边框，显示「备战席」标签

## 4. 转账

### TC-27 正常转账流程
- **操作：** 点击另一玩家头像 → 转账弹窗出现，收款方信息固定 → 输入 `5` → 点击「确认转账」
- **预期：** 我方分数 -5，对方分数 +5，生成记录「我 → 对方 5分」，弹窗关闭

### TC-28 转账金额为0或负数
- **操作：** 输入 `0` 或 `-5` → 确认
- **预期：** 不允许提交（按钮禁用或报错，金额须为正整数）

### TC-29 转账导致分数为负
- **前置：** 我方分数为 `10`
- **操作：** 向他人转账 `50`
- **预期：** 允许操作，我方分数变为 `-40`（无下限限制）

### TC-30 不能向自己转账
- **预期：** 自己的头像点击后不触发转账弹窗（或按钮不可用）

### TC-31 向备战席玩家转账
- **操作：** 点击备战席玩家头像 → 输入金额 → 确认
- **预期：** 正常转账，备战席玩家分数增加

### TC-32 转账弹窗关闭（点击遮罩）
- **操作：** 打开转账弹窗后点击弹窗外部遮罩
- **预期：** 弹窗关闭，无任何分数变动

## 5. 公池

### TC-33 交公池
- **操作：** 点击公池区域 → 点击「交公池（-1分）」
- **预期：** 我方分数 -1，公池分数 +1，生成记录，弹窗关闭

### TC-34 收公池（场上玩家）
- **前置：** 公池分数 > 0，操作者为场上玩家
- **操作：** 点击公池 → 点击「收公池（+X分）」
- **预期：** 我方分数 +X，公池归零，生成记录

### TC-35 公池为0时收公池按钮禁用
- **前置：** 公池分数为 0
- **预期：** 「收公池」按钮置灰不可点击

### TC-36 备战席玩家不可收公池
- **前置：** 操作者为备战席玩家，公池分数 > 0
- **预期：** 「收公池」按钮显示「备战席不可收公池」，置灰禁用

### TC-37 交公池导致分数为负
- **前置：** 我方分数为 0
- **操作：** 交公池
- **预期：** 允许，我方分数变为 -1，公池 +1

### TC-38 多人同时抢收公池（并发）
- **前置：** 公池分数 = 10，两名场上玩家几乎同时点击「收公池」
- **预期：** 只有一人成功收到公池，另一人收到失败提示（后端 RPC 原子操作保证）

## 6. 换座

### TC-39 进入换座模式
- **前置：** 操作者为房主
- **操作：** 点击顶部「换座」按钮
- **预期：** 顶部出现金色提示「换座模式 — 点击任意玩家选择」，所有玩家头像可点击

### TC-40 非房主看不到换座按钮
- **前置：** 操作者为普通玩家
- **预期：** 顶部不显示「换座」按钮

### TC-41 场上 ↔ 场上换座
- **操作：** 进入换座模式 → 点击左侧玩家 → 点击右侧玩家 → 确认
- **预期：** 两人 DB 座位互换，所有玩家视图同步更新，各自仍在自己的下方

### TC-42 场上 ↔ 备战席换座
- **操作：** 进入换座模式 → 点击一名场上玩家 → 点击一名备战席玩家 → 确认
- **预期：** 两人身份互换，原场上玩家进入备战席（失去收公池权限），原备战席玩家进入场上

### TC-43 换座到空位
- **前置：** 场上有空位
- **操作：** 进入换座模式 → 点击一名玩家 → 点击空位占位符 → 确认
- **预期：** 该玩家移至空位，原位置变为空位

### TC-44 房主在备战席时仍可换座
- **前置：** 房主已换至备战席
- **预期：** 顶部仍显示「换座」按钮，房主可正常发起换座操作

### TC-45 取消换座模式（未选人）
- **操作：** 进入换座模式 → 不点任何玩家 → 点击「取消」
- **预期：** 退出换座模式，无任何座位变动

### TC-46 取消已选中玩家
- **操作：** 进入换座模式 → 点击玩家A（高亮）→ 再次点击玩家A
- **预期：** 玩家A取消选中，可重新选择第一个玩家

### TC-47 房主与自己换座
- **操作：** 进入换座模式 → 点击自己（房主）→ 点击另一名玩家 → 确认
- **预期：** 房主与对方座位互换，房主标识（皇冠）跟随人，不跟随座位

## 7. 历史记录与撤回

### TC-48 记录列表显示
- **操作：** 点击底部「记录」
- **预期：** 按时间倒序显示所有交易记录，收入绿色，支出红色，公池相关金黄色

### TC-49 撤回操作
- **操作：** 点击某条记录的「撤回」→ 二次确认弹窗 → 确认
- **预期：** 原记录标记为「已撤回」（灰色），列表顶部新增一条【撤回】记录，相关分数反向恢复

### TC-50 已撤回记录不可再撤回
- **前置：** 已有一条「已撤回」状态的记录
- **预期：** 该记录无「撤回」按钮

### TC-51 【撤回】记录本身不可撤回
- **前置：** 已存在一条【撤回】操作记录
- **预期：** 该撤回记录无「撤回」按钮

### TC-52 撤回公池收取
- **前置：** 某玩家已收公池 10分
- **操作：** 撤回该笔记录
- **预期：** 该玩家分数 -10，公池 +10，生成撤回记录

### TC-53 撤回二次确认取消
- **操作：** 点击「撤回」→ 弹出确认框 → 点击取消
- **预期：** 弹窗关闭，记录和分数无变化

### TC-54 空记录列表
- **前置：** 房间刚创建，无任何交易
- **预期：** 记录弹窗显示空状态（无列表项），不报错

## 8. 房间设置

### TC-55 非房主进入设置
- **前置：** 操作者为普通玩家
- **操作：** 点击底部「设置」
- **预期：** 弹窗仅显示「退出房间」，不显示初始分数、踢人、重置、解散等选项

### TC-56 修改初始分数
- **前置：** 操作者为房主
- **操作：** 进入设置 → 将初始分数改为 `100` → 失焦
- **预期：** 自动保存，后续重置操作使用新的初始分数100

### TC-57 踢出玩家
- **操作：** 房主进入设置 → 点击玩家旁「踢出」→ 二次确认 → 确认
- **预期：** 被踢玩家立即跳回入口页，其座位变为空位，其他玩家视图更新

### TC-58 踢出二次确认取消
- **操作：** 点击「踢出」→ 弹出确认框 → 取消
- **预期：** 玩家未被踢出，弹窗关闭

### TC-59 重置分数
- **操作：** 房主点击「重置分数」→ 二次确认 → 确认
- **预期：** 所有玩家分数恢复至初始值，历史记录清空，所有人视图同步更新

### TC-60 解散房间
- **操作：** 房主点击「解散房间」→ 二次确认 → 确认
- **预期：** 所有玩家跳回入口页，房间状态变为 dissolved

### TC-61 退出房间（普通玩家）
- **操作：** 普通玩家点击「退出房间」
- **预期：** 本人跳回入口页，座位变为空位，其他人视图更新

### TC-62 房主退出 — 房间内有其他玩家
- **操作：** 房主点击「退出房间」
- **预期：** 提示将转让房主给谁（加入时间最早的玩家），确认后房主退出，该玩家获得房主权限和皇冠

### TC-63 房主退出 — 房间内只有房主一人
- **操作：** 房主（唯一玩家）点击「退出房间」
- **预期：** 提示房间将解散，确认后房间 dissolved，跳回入口页

## 11. 设备持久化

### TC-77 同设备重进同一房间
- **前置：** A设备已加入房间 `1234` 并进入主界面，之后关闭页面
- **操作：** A设备重新打开产品 → 进入加入房间 → 输入 `1234`
- **预期：** 跳过身份设置页，直接进入主界面，恢复之前的玩家身份和分数

### TC-78 被踢后重新加入
- **前置：** A设备被房主踢出
- **操作：** A设备尝试重新输入相同房间号进入
- **预期：** 跳转至身份设置页（本地身份记录已清除），需重新输入姓名

### TC-79 不同房间号互不干扰
- **前置：** A设备在房间 `1111` 有记录，在房间 `2222` 也有记录
- **操作：** 分别进入两个房间
- **预期：** 每个房间独立恢复对应身份，互不混淆

## 12. 房间生命周期
